## How to Submit
A Submission Package has to be uploaded on the [biodcase submission webpage](https://biodcase.github.io/challenge2026/submission) by accessing the link to the CMT submission system.
Additionally, the submission webpage contains the general description of the submission format and provides the technical report templates.
The required system output for the submission package is individual for each task and described in the following sections.

## System Output
The system output is usually generated by the participant's models on an evaluation dataset that is presented before the end of the competition.
In this task however, the evaluation dataset stays hidden and the participants have to submit their **submission models and inference code**.
For each model we automatically compute submission scores and rank the results for the competition.

## Inference Model Pipeline
Following files and folders are used for our inference pipeline:
```
root/
...
├── inference_handler.py - handles feature extraction and model
│
└── submission/
    ├── reports/ - generated, will contain your results
    ├── test_wav_files/ - test wav files on which your model will be tested
    ├── your_generated_code/ - (optional) place your generated code (src) here
    ├── your_submission_model/ - place saved model files here (.pth or .keras, .tflite)
    │
    ├── config_submission.yaml - config of your submission
    ├── submission_test.py - pre-test your submission
    └── <your_lastname>_task3_<x>.meta.yaml - template for system meta info
```
We implemented a inference model pipeline with `submission_test.py` where the `config_submission.yaml` has to be adapted to fit your model and feature extraction requirements handled by `./inference_handler.py`.
The file `config_submission.yaml` contains following:

```yaml
# inference handler - specify your submission model and feature extraction
inference_handler:

  # settings
  add_python_paths: []
  target_sample_rate: 24000
  target_wav_length_sec: 3
  model_dir: './your_submission_model'
  saved_model_extension: '.pth'
  tflite_model_extension: '.tflite'
  label_dict_yaml_path: './your_submission_model/label_dict.yaml'

  # feature handler
  feature_handler:
    module: feature_handler
    attr: FeatureHandler
    args: []
    kwargs: 
      target_sample_rate: 24000
      window_len: 4096
      window_stride: 512
      window_scaling_bits: 12
      mel_n_channels: 40
      mel_low_hz: 125
      mel_high_hz: 7500
      mel_post_scaling_bits: 6
      transpose_features_extracted: True
      normalize_features: True
      to_float: True
      add_channel_dimension: True
      add_batch_dimension: True
      channel_dimension_at_end: False

  # model
  model:
    module: pipeline_pytorch.model_tiny_ml
    attr: Baseline
    args: []
    kwargs: {}

# generate embedded code
generate_embedded_code:

  # template directory
  template_dir: '../biodcase_tiny/embedded/firmware'

  # gen code dir
  gen_code_dir: './your_generated_code'
  gen_code_source_folder_name: 'src'

  # replace with your serial device
  serial_device: '/dev/ttyUSB0'

  # quantize
  quantize: False
```
The fields you likely have to adapt are:
- "saved_model_extension" - Either '.pth' for pytorch or '.keras' for tensorflow
- "feature_handler"
  - "kwargs" - settings of your feature extraction algorithm
- "model"
  - "module" - location of your model class
  - "attr" - model class name

After you have set `config_submission.yaml` you can evaluate if your model successfully runs by:
```bash
python submission_test.py
```

## Submission Package Format
Similar to the [biodcase submission webpage](https://biodcase.github.io/challenge2026/submission) we require a `.zip` package with following structure:
```
biodcase_2026_task3_submission_package.zip/
├── <your_lastname>_task1.technical_report.pdf
├── <your_lastname>_task3_1/
│   ├── <your_lastname>_task3_1.meta.yaml
│   ├── config_submission.yaml
│   ├── <replacement or additional files>
│   ... 
│   ├── your_submission_model/
│   │   └── <model_name>.pth or <model_name>.keras 
│   │   └── <model_name>.tflite (optional)
│   │
│   └── your_generated_code/
│        └── src (optional, but appreciated)
│
├── <your_lastname>_task3_2/ (optional)
│    ├── ...
...       
```
where the `<replacement or additional files>` are custom files of participants either to replace an existing file by name or by adding a non-exiting file to the root directory of the test repository.
