Description of submissions to the BioDCASE-Tiny 2026 competition (Task 3)

## How to Submit
Submissions have to be done on the [biodcase submission webpage] (https://biodcase.github.io/challenge2026/submission) by accessing the link to the CMT submission system.
Additionaly, the submission webpage contains the general description of the submission format and the technical report templates.
The system output is for each task different and described in the following section.

## System Output
The system output is usually generated by the participant's models on an evaluation dataset that is presented before the end of the competition.
In this task however, the evaluation dataset stays hidden and the participants have to submit their submission models.
For each model we automatically compute submission scores and rank the results for the competion.

## Submission Package Format
Similar to the [biodcase submission webpage] (https://biodcase.github.io/challenge2026/submission) we require a `.zip` package with following structure:
```
biodcase_2026_task3_submission_package.zip/
├── <your_lastname>_task1.technical_report.pdf
├── <your_lastname>_task3_1/
│   ├── <your_lastname>_task3_1.meta.yaml
│   ├── config_submission.yaml
│   ├── <replacement or additional files>
│   ... 
│   ├── your_submission_model/
│   │   └── <model_name>.pth or <model_name>.keras 
│   │   └── <model_name>.tflite (optional)
│   │
│   └── your_generated_code/
│        └── src (optional, but appreciated)
│
├── <your_lastname>_task3_2/ (optional)
│    ├── ...
...       
```
where the `<replacement or additional files>` are custom files of participants either to replace an existing file by name or by adding a non-exiting file to the root directory of the test repository.
